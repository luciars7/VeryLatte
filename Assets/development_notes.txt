    Things are going to be as follows:

        -- Controllers will be managed with a ControllerManager.
            -> This will tell the GameManager (with a call to a function), which object has just been picked up.
        -- Screen showing orders will have a ScreenManager.
            -> It will show which object is to be picked up next.
            -> In GameManager delegate-event mechanism can be used to tell the screen to change the picture for the new order.
            In GameManager it has to be checked if picked up object was the right one. In case it is right, pic will change,
            in case it is not, pic won't change.
        -- Foods will have a FoodManager.
            -> It will activate an animation that will make the object disappear from its place and go somewhere else.
